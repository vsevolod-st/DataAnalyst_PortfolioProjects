# Анализ данных онлайн библиотеки

## Описание проекта
В этом проекте будем анализировать данные онлайн библиотеки - сервиса для чтения электронных книг и прослушивания аудиокниг по подписке.
Нам предстоит рассчитать несколько метрик с помощью SQL, а затем проверить гипотезу с помощью Python и составить аналитическую записку.

## Задачи проекта
**Часть 1.** [Расчет ключевых метрик онлайн библиотеки](./SQL_Metrics.sql):

В этой части проекта с помощью SQL рассчитаем несколько ключевых метрик активности пользователей сервиса. Цель — глубже понять поведение пользователей и найти примечательные закономерности. Эти расчёты помогут понять, как пользователи взаимодействуют с платформой, какие виды контента наиболее популярны, а также выявить особенности поведения пользователей.

У нас в распоряжении имеются данные о чтении и прослушивании контента в сервисе онлайн-библиотеки, которые включают информацию о пользователях, платформах, времени, длительности сессий и типах контента. Данные представлены за период с 1 сентября по 11 декабря 2024 года.

**Часть 2.** [Проверка продуктовой гипотезы](./Online_Library_Hypothesis.ipynb):

При расчёте ключевых метрик сервиса было определено, что у пользователей из Санкт-Петербурга LTV выше, чем у пользователей из Москвы.

Команда сервиса онлайн-библиотеки изучила данные, полученные при анализе LTV, и выдвинула гипотезу. Она заключается в том, что пользователи, которые приносят больший LTV, являются более лояльными, а значит, проводят в среднем больше времени за чтением и прослушиванием книг в приложении.

В этой части проекта мы сравним пользователей из Санкт-Петербурга и Москвы и проверим гипотезу о том, что пользователи из Санкт-Петербурга проводят в среднем больше времени за чтением и прослушиванием книг в приложении по сравнению с пользователями из Москвы.
